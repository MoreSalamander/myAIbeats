# my-AI-beats — CONSTITUTION

The non-negotiable laws of the project. Edit this before changing any
dependent code. Doctrine-first, per the MoreSalamander thesis.

---

## Article I — The SongSpec is the constraint document

A `SongSpec` is authored (or generated by my-AI-script) before any model
runs. It declares everything the song must contain: title, genre, tempo,
key, mode, time signature, mood, energy arc, instruments, and a prompt
for every section. The models work inside the spec. They never edit it.

## Article II — The hidden gem: musical continuation

Each section after the first is synthesized using MusicGen's audio
continuation mode — the last few seconds of the previous section are
passed as conditioning audio. Sections don't just stitch; they flow.
This is the feature that makes my-AI-beats sound like a song rather
than a playlist of unrelated clips.

## Article III — The model proposes; Python disposes

Every generated section crosses two gates before it is accepted:
- `section_verify` — CLAP scores the audio against the section's prompt.
  The audio must sound like what was asked for.
- `duration_verify` — the audio must match the declared bar count
  (±tolerance). Wrong length means wrong structure.

Both are blocking. The section retries within MAX_RETRIES. After
exhaustion it falls back to a neutral tone pad — the song continues,
never corrupts.

## Article IV — Vocals are enhancement; the instrumental is the premise

The vocal layer is non-blocking by design. A vocal that fails synthesis,
quality check, or alignment is dropped. The instrumental track ships
regardless. Silence is preferable to a broken vocal that undermines the
music.

## Article V — Energy is a first-class citizen

Every section declares an `energy` value (0.0–1.0). The pipeline uses
this to parameterize the synthesis prompt automatically — it is not
decoration. An intro at 0.2 and a chorus at 0.9 produce audibly
different outputs from the same base prompt. The energy arc is the
song's emotional shape.

## Article VI — Bounded retry, defined fallback

No unbounded loops. Each synthesis stage retries up to MAX_RETRIES,
then takes a named fallback: a neutral tone pad for sections, silence
for vocals. Thrashing is a bug. Every fallback emits an event.

## Article VII — Offline-provable before online-expensive

The entire pipeline must run with zero model downloads using
`ScriptedRenderer`. All gates fire. All events emit. All section
manifests are produced. The scaffold is proven before any weight loads.

## Article VIII — One observable pipeline, shared vocabulary

Every stage emits the studio's shared NDJSON event vocabulary:
`step_start` / `step_complete` / `gate_pass` / `gate_fail` / `retry`
/ `fallback` / `skip` / `done` / `error`. Same vocabulary as every
other tool in the my-AI suite.
