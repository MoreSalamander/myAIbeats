my-AI-beats

A local music generator. A SongSpec becomes a full song — local MusicGen renders each section, a CLAP gate scores each against its intended vibe, and ffmpeg stitches the survivors into one track. No cloud, no API keys.

A spec in, a song out — and the gate has ears.

What it does

Sectioned, continued, scored, stitched.

You author a SongSpec — sections, style, mix, optional vocals — validated by spec.py before a note is rendered (a structural failure is an authoring bug, never a model failure). Then the pipeline generates the song section by section and keeps only what actually sounds like what you asked for.

SongSpec  →  MusicGen sections (w/ continuation)  →  CLAP gate scores each  →  ffmpeg stitch  →  song.wav
The hidden gem: audio continuation. Each section is generated as a continuation of the last, so the song flows as one piece instead of a playlist of disconnected clips — then a CLAP model judges each section against its prompt, the deterministic ear that keeps a wandering generation honest.

How it works

The model proposes audio; a model-free contract and a CLAP gate dispose.
  1. Validate the spec. spec.py checks the SongSpec is complete and renderable before anything runs.
  2. Render sections. MusicGen generates each section, continuing the previous one for coherence.
  3. Score with CLAP. Each section is scored against its intended vibe; the ones that miss are flagged or regenerated.
  4. Stitch. ffmpeg assembles the kept sections into one continuous song.wav.
